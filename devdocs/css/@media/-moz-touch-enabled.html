@media.-moz-touch-enabled

Non-standard
This feature is non-standard and is not on a standards track. Do not use it on production sites facing the Web: it will not work for every user. There may also be large incompatibilities between implementations and the behavior may change in the future.

Note: Since Firefox 58, this media feature is no longer available to web content â€” it is only available internally (e.g. to XUL / chrome code). See bug 1396066 for more details.

Syntax

<integer>
If the device supports touch events (for a touch screen), this is 1. Otherwise it's 0.

Media: media/visual
Accepts min/max prefixes: no

Example

You might use this feature to render your buttons slightly larger if the user is on a touch-screen device, to make them more finger-friendly.

button {
  padding: .5em;
}

@media (-moz-touch-enabled) {
  button {
    padding: 1em;
  }
}

See also

  • This feature corresponds to the :-moz-system-metric(touch-enabled) CSS pseudo-class.
